Types of Authentication

In-Memory Authentication

The module supports in-memory authentication by entering the user, password and role details in the overrided configure method of SpringSecurityConfig.java class. By default the passwords entered as a string are encoded. You can modify this class to add or remove users depending on your needs. In-memory authentication can be implemented in two ways.

  • Hardcoding the acceptable values for usernames, their corresponding passwords and roles.
  • Using in-built databases to fetch the user details for verification.



OAuth 2.0 Authentication

Firstly, to add OAuth Authentication, you need to add the following dependencies in pom.xml file.

pom.xml 

                    <dependency>
                            <groupId>org.springframework.boot</groupId>
                            <artifactId>spring-boot-starter-oauth2-client</artifactId>
                    </dependency>

We can provide OAuth authentication for various social logins like Google, LinkedIn, GitHub, Facebook etc. For the purpose of understanding, we have implemented only Sign-In with Google. Others can be implemented in a similar way.

For Google sign-in, you need to create a client-id and client-secret for Google OAuth 2.0 and enter them in application.properties file. The Google Sign-in option is available along with the sign-in form for alternate authentication.




Hibernate Authentication

AuthModule is a pluggable module as it integrates Hibernate along with Spring Security to make it database independent. Since Hibernate is an Object-Relational Mapping (ORM) tool, it maps to any database using an object of CustomUserDetails.java class. The server details and credentials for the database can be added in the application.properties file and the configuration for Hibernate can be specified in hibernate.config.xml file.
